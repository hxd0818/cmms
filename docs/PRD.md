# CMMS 会务管理系统 — 产品需求文档 (PRD)

> Conference Management & Member Service
> 版本: v1.0 | 更新: 2026-07-18

---

## 1. 产品概述

### 1.1 定位

CMMS 是一款**以接待运营为核心**的通用会务管理平台。覆盖从会前筹备到会后总结的全链路：嘉宾管理、议程编排、接送调度、住宿分配、餐饮安排、礼品发放、接待陪同、费用统计。

### 1.2 核心差异

- **会议独立资源池**：每场会议管理自己的车辆、酒店房间、餐桌，互不干扰
- **跨会议嘉宾沉淀**：VIP 档案长期累积，同一嘉宾参加多场会议只存一份
- **三端门户**：工作人员端（PC/H5）+ 嘉宾端（H5 链接）+ 司机端（H5 链接）+ 接待端（H5 链接）
- **字典后台可配置**：所有枚举标签集中管理，管理员改标签不需要改代码
- **状态机驱动**：每个业务环节都有严格的状态转换规则

### 3.5 会议嘉宾

#### 3.5.1 主从关系

- 主嘉宾（PRIMARY）：独立参会
- 随行人员：关联到主嘉宾，角色包括：秘书/安保/翻译/家属/助理/司机
- 规格继承：随行可继承主嘉宾的接送（inheritTransport）和住宿（inheritLodging）
- 等级覆盖：可单独设置 levelOverride

#### 3.5.2 功能

- 搜索添加嘉宾（从全局嘉宾库）
- 批量导入（Excel：姓名/手机/分组/角色/主嘉宾手机/等级覆盖）

- RSVP 状态管理（待确认/已确认/已拒绝）
- 签到状态管理
- 分享嘉宾端行程链接
- 点击嘉宾行 → 侧滑面板查看该嘉宾的全部接待任务

#### 3.5.3 接待签到状态机

```
NOT_ARRIVED → CHECKED_IN → IN_HOUSE → DEPARTED
     ↓
  NO_SHOW (终态)
```

- 完成时自动生成费用记录

- 按车牌/司机/车队/车型搜索

## 4. 三端门户

### 4.1 嘉宾端 `/guest/[token]`

- **认证**：HMAC-SHA256 Token（30 天有效，可吊销）
- **内容**：行程总览（接送/住宿/餐饮/议程/礼品/接待人员）
- **RSVP**：显示参会状态
- **无需登录**

### 4.2 司机端 `/driver/[token]`

- **认证**：HMAC Token（7 天有效）
- **内容**：当日同车全部任务列表（当前任务高亮）
- **操作**：出发 → 接到 → 送达（状态自更新）
- **拼车可见**：同车其他任务信息

### 4.3 接待端 `/companion/[token]`

- **认证**：Companion ID（CUID）
- **内容**：分配到的每位嘉宾的完整接待信息
  - 嘉宾档案（等级/单位/职务/电话/饮食禁忌）
  - 接送安排（路线/时间/车辆/司机）
  - 住宿（酒店/房号/日期）
  - 用餐（餐类/餐桌/时间）
  - 会议议程（全部议程项）
- **分享**：可为每位嘉宾生成行程链接

## 6. 安全与权限

### 6.1 认证

- NextAuth v5（Credentials Provider）
- JWT Session（8 小时）
- Dual Config（Edge-safe + Node 完整）

### 6.2 权限

- CASL：SUPER_ADMIN（全部管理）/ VIEWER（全部只读）
- 会议级角色：MeetingStaff 表（10 种角色）

### 6.3 数据安全

- 手机/身份证：AES-256-GCM 加密（Prisma $extends 透明加解密）
- VIEWER 角色：字段脱敏（138****1234）
- Token：HMAC-SHA256 哈希存储（原值不入库）

### 6.4 速率限制

- 登录端点：5 次/5 分钟（Redis 滑动窗口）
- Fail-open 设计（Redis 不可用时不阻塞登录）

### 6.5 审计

- 48 个日志点覆盖全部 Server Action
- fire-and-forget（prisma.auditLog.create + .catch）

### 7.2 领域分层

- `service.ts` — 业务规则（错误处理 + 状态机 + 冲突检测）

Server Actions（`app/actions/`）统一入口：auth → validation → service → audit → revalidate。

## 10. 业务流程总览

```
创建会议 (DRAFT)
    │
    ├── 分配工作人员角色
    ├── 添加嘉宾（单个/批量导入）
    │     └── 设置主从关系 / 等级覆盖 / 继承标志
    ├── 编排议程
    │     └── 演讲嘉宾冲突检测
    ├── 管理资源（车辆/酒店/餐桌）
    ├── 创建接待任务
```
